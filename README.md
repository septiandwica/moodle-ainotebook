# DEMI Tutor (AI Notebook)

DEMI Tutor (AI Notebook) is an advanced, premium AI-powered study assistant plugin for Moodle. Developed by **septiandwica** and supported by **Tateta** ([samastanuswantara.com](https://samastanuswantara.com)). This plugin transforms traditional study materials into interactive, AI-driven learning experiences with a completely customized and premium UI.

![PresMate Icon](pix/icon.svg)

## 🚀 Key Features

### 🧠 Intelligent Study Chat
- **Multi-File Context Selection**: Users can select specific PDF or text materials using checkboxes. The AI dynamically answers questions based *only* on the selected context, giving users total control over their learning scope.
- **Context-Aware RAG (Retrieval-Augmented Generation)**: Extracts text from PDFs using `pdftotext` and `tesseract` OCR for scanned images, feeding them as context.
- **Chat History Per User**: Maintains persistent chat history context for each user, allowing for natural, multi-turn conversations.
- **Configurable Personalities**: Users can adjust the AI's Tone (*Helpful Assistant*, *Professional Tutor*, or *Critical Thinker*) and Length (*Brief*, *Balanced*, *Detailed*) via the settings menu.
- **Dynamic Suggestions**: Automatically generates 3 smart follow-up questions tailored to the student's current chat context and material.

### 🎨 Creator Hub (Learning Artifacts)
Generates professional-grade academic materials inside a dedicated Creator Hub workspace:
- **Interactive Quizzes**: Generates 4-option multiple-choice questions parsed directly into an interactive UI. Features instant scoring, hints, feedback, and **Moodle Gradebook Integration** (automatically syncs quiz scores to the course grades).
- **Visual Mindmaps**: Automatically generates and renders diagrams using Mermaid.js to visualize complex concepts. Includes an automatic syntax sanitizer to fix common AI formatting mistakes.
- **Study Reports**: Generates detailed, structured markdown reports summarizing key takeaways.
- **Save & Manage**: Students can save generated artifacts to the database to revisit later, or delete them.

### 👨‍🏫 Teacher Dashboard & Analytics
- **AI-Powered Insights**: The AI acts as an *Academic Evaluator*, automatically reading a student's chat history and generated artifacts to determine their level of understanding, summarizing their activity, and providing actionable recommendations for the teacher.
- **Automated Scoring**: The AI evaluates the depth of the student's questions and assigns a score (0-100) that is pushed to the Moodle Gradebook.
- **View Workspace**: Teachers can click "View Workspace" to enter a Read-Only mode of any student's workspace, allowing them to see the exact chat log and artifacts generated by the student.
- **Reminders**: Teachers can send instant Moodle notifications to inactive students with one click.

### 📄 Professional PDF Export
- **Clean Layouts**: Export your generated reports, quizzes, and mindmaps into high-quality PDFs directly from the UI.
- **Branded Headers**: Every export dynamically includes the formal university header, custom logo, and student metadata.

### 💎 Premium UI/UX
- **Modern Aesthetics**: Built with a sleek glassmorphism design, vibrant color palettes, tailored typography, and smooth micro-animations.
- **Split Workspace**: Independent scrolling and isolated preview areas for the chat and the Creator Hub.

### ⚙️ Multi-Provider Support
- **Moodle Core AI Subsystem**: Fully supports Moodle's native AI provider API.
- **Custom Providers**: Built-in direct support for Groq, OpenAI, and Gemini APIs with custom model selection if Moodle's core AI is not used.

## 🧩 Architecture & Workflow (How It Works)

The plugin follows a sophisticated Retrieval-Augmented Generation (RAG) pipeline to ensure responses are accurate, fast, and strictly bounded to the academic materials.

1. **Input & Context Selection (`view.php` ➔ `chat_ajax.php`)**
   - The student types a question and selects which study files (PDFs/Text) they want the AI to read.
   - The UI sends an AJAX request containing the prompt, chat configurations, and the `file_ids` to the backend.

2. **Context Extraction & Caching (`classes/ai_client.php`)**
   - The system checks if the text for the selected files is already cached.
   - If not, it uses system binaries (`pdftotext`) to parse the PDF layout. For scanned documents, it falls back to Optical Character Recognition (OCR) using `tesseract` (supporting English and Indonesian).
   - The extracted raw text is cached in Moodle's Universal Cache (MUC) to ensure fast subsequent responses.

3. **Prompt Engineering & Assembly**
   - The system dynamically builds a massive prompt containing:
     - **System Directives**: Instructions on the AI's persona, toxicity filters, and the strict rule to reject out-of-context questions.
     - **Chat History**: The 5 most recent interactions are pulled from the `ainotebook_chat` database table to maintain conversational context.
     - **Material Context**: The extracted text from the selected files.
     - **User Request**: The student's actual question.

4. **LLM Processing & Routing**
   - The assembled prompt is routed to the configured provider (Moodle Core AI Subsystem, Groq, Gemini, or OpenAI).
   - The LLM processes the context and returns the generated content.

5. **Sanitization & Parsing**
   - The response is intercepted by the backend parser.
   - If the user requested a **Mindmap**, the system runs the generated Mermaid syntax through a custom regex sanitizer (`sanitize_mermaid`) to fix common AI formatting errors (like unquoted labels or broken arrows) that would otherwise crash the renderer.
   - The raw response is then returned to the frontend.

6. **UI Rendering & Separation**
   - The frontend JavaScript (`view.php`) receives the response.
   - It automatically detects if the response contains an artifact (e.g., ````json-quiz````, ````mermaid````, or `[REPORT_START]`).
   - Artifacts are stripped from the main chat window and rendered beautifully inside the interactive **Creator Hub** panel, while conversational text remains in the chat.

## 🛠 Installation

1. Clone this repository into your Moodle installation:
   ```bash
   git clone https://github.com/septiandwica/moodle-ainotebook.git mod/ainotebook
   ```
2. Log in to your Moodle site as an administrator and go to **Site Administration > Notifications**.
3. Follow the prompts to install the plugin.
4. Configure the AI settings (API keys, provider, prompts) in **Site Administration > Plugins > Activity Modules > AI Notebook**.

## ⚙️ Technical Requirements

- **Moodle**: 4.1+ / 5.0+
- **System Binaries**: `pdftotext` (poppler-utils) and `tesseract-ocr` (with eng+ind language packs) installed on the server for PDF extraction and OCR fallback.
- **Dependencies**: 
  - [Mermaid.js](https://mermaid.js.org/) for mindmaps.
  - [Marked.js](https://marked.js.org/) for markdown rendering.

## 🔒 Security & Privacy

- **Toxicity Filter**: Built-in system prompt guards against prompt injection and toxic language.
- **Context Locking**: The AI is strictly instructed to stick to the provided study materials and reject out-of-context academic questions.
